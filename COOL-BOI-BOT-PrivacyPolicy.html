Privacy declarations for COOL BOI BOT

Message and data collection from COOL BOI BOT:

Message data such as content and members are checked and stored for the leveling system. These pieces of data are unreadable by humans and thus we never check it. Only message number is stored, other data such as content, member, guild, and other are cached temporarily for commands such as user-info and server and many more commands use this data.

Presence and member data collection from COOL BOI BOT:

Presence data such as online status are stored in an unreadable cache, used in certain commands to display presence information. Other data such as activity, member updates (for presence data), and others are NOT collected, stored, or shared in any way.

Server member list data collection from COOL BOI BOT:

Server members can have valuable data, it's the reason why it is a “Privileged Intent” in discord developer portal. This data is used all over the bot, in logging, in general commands, and many more. This also gives us user updates such as when a user joined or left a server, and all the other updates. We use this to create welcome and goodbye messages on a server. Like other data logging, this is configurable using the commands in the “util” category. This data is stored in an unreadable cache, but also sent in some messages publicly. We DO NOT share any of the massive and useful data to anyone, it is stored and contained within the bot.

Event data collection from COOL BOI BOT:

Event data can be handled by the developer via Intents, by default all of these are enabled, meaning that we’ll get every event that the discord gateway provides. These data can provide various and massive amounts of data. If you want to see this, we have a feature called “logging”. What logging does is it makes it easier for server moderators to see who did what. AN example  of this is who deletes a message? WHat did the message contain? All of that stuff. Since these are sensitive data to some, it can be disabled by using setLogChannel and setting it to “none” to disable it. When enabled data is cached, and sent to the specified channel. It is sent nowhere else but there.

Other data:

We mentioned all of the sensitive and useful data here. Other data is probably stored in an unreadable cache, so you can rest easy on other items we didn’t mention

Anyways thank you for using COOL BOI BOT.
